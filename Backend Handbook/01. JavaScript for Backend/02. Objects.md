# Objects

---

# Why

Objects are one of the most important data structures in JavaScript.

Backend applications constantly work with objects because almost everything is represented as an object.

Examples include:

- User data
- Blog data
- Request data (`req`)
- Response data (`res`)
- Database documents
- JSON data
- Configuration values

Understanding objects is essential for backend development.

---

# What

An object is a collection of related data stored as **key-value pairs**.

Syntax

```js
const user = {
    username: "Kirat",
    age: 21,
    isAdmin: false
};
```

Here,

- `username`, `age`, and `isAdmin` are **keys**.
- `"Kirat"`, `21`, and `false` are **values**.

---

# Object Structure

```js
const object = {
    key: value,
    key: value
};
```

Example

```js
const blog = {
    title: "Learning Backend",
    description: "Express and MongoDB",
    author: "Kirat"
};
```

---

# Accessing Object Properties

## Dot Notation

```js
const user = {
    username: "Kirat"
};

console.log(user.username);
```

Output

```text
Kirat
```

---

## Bracket Notation

```js
const user = {
    username: "Kirat"
};

console.log(user["username"]);
```

Output

```text
Kirat
```

Both methods return the same value.

---

# Dot vs Bracket Notation

Use **dot notation** when you know the property name.

```js
user.username
```

Use **bracket notation** when the property name is stored inside a variable.

```js
const key = "username";

console.log(user[key]);
```

This is called **dynamic property access**.

---

# Adding New Properties

Objects can be updated at any time.

```js
const user = {
    username: "Kirat"
};

user.email = "kirat@gmail.com";
```

Result

```js
{
    username: "Kirat",
    email: "kirat@gmail.com"
}
```

---

# Updating Properties

```js
const user = {
    username: "Kirat"
};

user.username = "Singh";
```

Result

```js
{
    username: "Singh"
}
```

---

# Deleting Properties

```js
const user = {
    username: "Kirat",
    age: 21
};

delete user.age;
```

Result

```js
{
    username: "Kirat"
}
```

---

# Nested Objects

Objects can contain other objects.

```js
const user = {
    username: "Kirat",

    address: {
        city: "Bathinda",
        state: "Punjab"
    }
};
```

Access

```js
user.address.city
```

Output

```text
Bathinda
```

# Real Project Usage

### Reading Request Data

```js
const title = req.body.title;

const description = req.body.description;
```

---

### Reading Route Parameters

```js
const blogId = req.params.id;
```

---

### Reading Logged-in User

```js
const userId = req.PrcUser._id;
```

---

### Creating API Response

```js
return res.status(200).json({
    message: "Success",
    data: blog
});
```

---

### Building Update Object

```js
let updateData = {};

updateData.title = req.body.title;
updateData.description = req.body.description;
```

# Common Mistakes

### Mistake 1

Using dot notation with variables.

Wrong

```js
const key = "username";

user.key
```

Result

```text
undefined
```

Correct

```js
user[key]
```

---

### Mistake 2

Confusing keys and values.

```js
{
    username: "Kirat"
}
```

Key

```text
username
```

Value

```text
Kirat
```

---

### Mistake 3

Trying to access a property that doesn't exist.

```js
console.log(user.phone);
```

Output

```text
undefined
```

---

# Best Practices

- Use meaningful property names.
- Prefer dot notation when the property name is known.
- Use bracket notation for dynamic property access.
- Keep object structure clean and organized.
- Avoid deeply nested objects unless necessary.

---

# Summary

- Objects store data as key-value pairs.
- Backend development relies heavily on objects.
- Properties can be accessed using dot or bracket notation.
- Objects can be updated, extended, and nested.
- `req.body`, `req.params`, database documents, and API responses are all objects.
