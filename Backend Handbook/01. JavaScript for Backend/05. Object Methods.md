# Object Methods

---

# Why

Objects are widely used in backend development.

Sometimes we need to:

- Get all keys
- Get all values
- Get both keys and values
- Check if an object is empty
- Loop through an object dynamically

JavaScript provides built-in methods that make these tasks simple.

---

# What

The most commonly used object methods in backend development are:

- `Object.keys()`
- `Object.values()`
- `Object.entries()`

These methods convert an object into an array, making it easier to work with.

---

# Object.keys()

Returns an array containing all the keys of an object.

Syntax

```js
Object.keys(object)
```

Example

```js
const user = {
    username: "Kirat",
    age: 21,
    city: "Bathinda"
};

const keys = Object.keys(user);

console.log(keys);
```

Output

```js
["username", "age", "city"]
```

---

# Object.values()

Returns an array containing all the values of an object.

Syntax

```js
Object.values(object)
```

Example

```js
const user = {
    username: "Kirat",
    age: 21,
    city: "Bathinda"
};

const values = Object.values(user);

console.log(values);
```

Output

```js
["Kirat", 21, "Bathinda"]
```

---

# Object.entries()

Returns an array of key-value pairs.

Syntax

```js
Object.entries(object)
```

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

const entries = Object.entries(user);

console.log(entries);
```

Output

```js
[
    ["username", "Kirat"],
    ["age", 21]
]
```

---

# Checking if an Object is Empty

One common backend task is checking whether `req.body` contains any data.

Example

```js
if (Object.keys(req.body).length === 0) {
    return res.status(400).json({
        message: "Request body is empty"
    });
}
```

Explanation

```text
Object.keys(req.body)

↓

Returns an array of keys

↓

.length

↓

If length is 0

↓

Object is empty
```

# Looping Through Object Keys

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

for (const key of Object.keys(user)) {
    console.log(key);
}
```

Output

```text
username
age
```

---

# Looping Through Object Values

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

for (const value of Object.values(user)) {
    console.log(value);
}
```

Output

```text
Kirat
21
```

---

# Looping Through Object Entries

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

for (const [key, value] of Object.entries(user)) {
    console.log(key, value);
}
```

Output

```text
username Kirat
age 21
```

# Real Project Usage

### Example 1

Checking if request body is empty.

```js
if (Object.keys(req.body).length === 0) {

}
```

---

### Example 2

Allowing only specific fields.

```js
const allowedFields = ["title", "description"];

let updateData = {};

for (const field of allowedFields) {

    if (field in req.body) {
        updateData[field] = req.body[field];
    }

}
```

---

### Example 3

Getting all keys from request body.

```js
const keys = Object.keys(req.body);

console.log(keys);
```

---

### Example 4

Dynamic property access.

```js
const key = "title";

console.log(req.body[key]);
```

# Common Mistakes

### Mistake 1

Thinking `Object.keys()` returns values.

Wrong

```js
Object.keys(user);
```

Output

```text
["username", "age"]
```

Not

```text
["Kirat", 21]
```

---

### Mistake 2

Using dot notation with dynamic keys.

Wrong

```js
const key = "username";

user.key
```

Correct

```js
user[key]
```

---

### Mistake 3

Forgetting that these methods return arrays.

Example

```js
const keys = Object.keys(user);

console.log(Array.isArray(keys));
```

Output

```text
true
```

---

# Best Practices

- Use `Object.keys()` when working with object properties.
- Use `Object.values()` when only values are required.
- Use `Object.entries()` when both keys and values are needed.
- Use `Object.keys(obj).length` to check if an object is empty.
- Combine these methods with `for...of` for clean and readable code.

---

# Summary

- `Object.keys()` returns an array of keys.
- `Object.values()` returns an array of values.
- `Object.entries()` returns key-value pairs.
- These methods are heavily used in backend development for validation, dynamic updates, and object iteration.
- `Object.keys(req.body).length === 0` is a common way to check if a request body is empty.
