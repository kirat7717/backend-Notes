# Destructuring

---

# Why

In backend development, we often work with objects such as:

- `req.body`
- `req.params`
- `req.query`
- `process.env`
- Database documents

Without destructuring, we repeatedly write the object name to access each property.

Destructuring provides a cleaner and shorter way to extract values from objects and arrays.

---

# What

Destructuring is a JavaScript feature that extracts values from objects or arrays and stores them in variables.

---

# Object Destructuring

Without Destructuring

```js
const username = req.body.username;
const email = req.body.email;
const password = req.body.password;
```

With Destructuring

```js
const { username, email, password } = req.body;
```

Both approaches produce the same result.

---

# Syntax

```js
const { property1, property2 } = object;
```

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

const { username, age } = user;
```

---

# Renaming Variables

Sometimes you want a different variable name.

```js
const user = {
    username: "Kirat"
};

const { username: name } = user;

console.log(name);
```

Output

```text
Kirat
```

---

# Default Values

If a property doesn't exist, a default value can be provided.

```js
const user = {
    username: "Kirat"
};

const { city = "Bathinda" } = user;
```

Output

```text
Bathinda
```

---

# Array Destructuring

Arrays can also be destructured.

```js
const colors = ["Red", "Green", "Blue"];

const [first, second] = colors;
```

Output

```text
first = Red
second = Green
```

---

# Skipping Elements

```js
const colors = ["Red", "Green", "Blue"];

const [, second] = colors;
```

Output

```text
Green
```

---

# Nested Destructuring

Objects can contain other objects.

```js
const user = {
    username: "Kirat",

    address: {
        city: "Bathinda"
    }
};

const {
    address: { city }
} = user;
```

Output

```text
Bathinda
```

---

# Destructuring in Backend

Destructuring is used throughout backend development.

Examples include:

- Request body
- Route parameters
- Query parameters
- Environment variables
- Database documents

---

# Real Project Usage

### Reading Request Body

Without Destructuring

```js
const title = req.body.title;
const description = req.body.description;
```

With Destructuring

```js
const { title, description } = req.body;
```

---

### Reading Route Parameters

Without Destructuring

```js
const blogId = req.params.id;
```

With Destructuring

```js
const { id } = req.params;
```

---

### Reading Query Parameters

```js
const { page, limit } = req.query;
```

---

### Reading Environment Variables

```js
const { PORT, MONGO_URI } = process.env;
```

---

### Reading Logged-in User

```js
const { _id } = req.PrcUser;
```

# Advantages

- Less code.
- Easier to read.
- Cleaner controllers.
- Reduces repetition.
- Makes object handling simpler.

---

# Common Mistakes

### Mistake 1

Using the wrong property name.

Wrong

```js
const { name } = req.body;
```

If `req.body` contains `username`, the value will be `undefined`.

Correct

```js
const { username } = req.body;
```

---

### Mistake 2

Destructuring from `undefined`.

Wrong

```js
const { title } = undefined;
```

This throws an error.

Always ensure the object exists.

---

### Mistake 3

Thinking destructuring changes the original object.

```js
const user = {
    username: "Kirat"
};

const { username } = user;
```

The original object remains unchanged.

---

# Best Practices

- Use destructuring when extracting multiple properties from an object.
- Use meaningful variable names.
- Use default values when appropriate.
- Avoid unnecessary nested destructuring if it reduces readability.

---

# Summary

- Destructuring extracts values from objects and arrays.
- It reduces repetitive code.
- Backend developers frequently use it with `req.body`, `req.params`, `req.query`, and `process.env`.
- Destructuring improves readability and makes controllers cleaner.
