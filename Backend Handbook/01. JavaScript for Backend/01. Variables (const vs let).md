# Variables (const vs let)

---

# Why

Variables are used to store data in memory so it can be used later in the program.

Almost every backend application stores values such as:

- User data
- Request data
- Database results
- Tokens
- IDs
- Configuration values

Without variables, we would have to write the same values repeatedly.

---

# What

JavaScript provides different ways to declare variables.

The two most commonly used in modern backend development are:

- `const`
- `let`

Backend developers rarely use `var`.

---

# const

`const` creates a variable whose reference cannot be reassigned.

```js
const username = "Kirat";
```

Valid

```js
const age = 21;
const isLoggedIn = true;
```

Invalid

```js
const age = 21;

age = 22; // ❌ Error
```

A `const` variable must always be initialized when it is created.

```js
const name = "Kirat";
```

This is invalid.

```js
const name; // ❌ Error
```

---

# let

`let` creates a variable whose value can be changed later.

```js
let count = 0;

count = 1;
count = 2;
```

This is useful when the value changes during program execution.

---

# const vs let

| const | let |
|--------|-----|
| Cannot be reassigned | Can be reassigned |
| Must be initialized immediately | Initialization is optional |
| Preferred by backend developers | Used only when reassignment is required |

---

# Important Point

`const` does **not** make an object or array immutable.

It only prevents reassignment of the variable.

Example:

```js
const user = {
    name: "Kirat"
};

user.name = "Singh"; // ✅ Allowed

user.age = 21; // ✅ Allowed
```

But this is not allowed.

```js
const user = {
    name: "Kirat"
};

user = {}; // ❌ Error
```

The reference cannot change.

The contents can.

---

# When to Use const

Use `const` when the variable should always refer to the same value or object.

Examples

```js
const app = express();

const user = await User.findById(id);

const token = jwt.sign(payload, secret);

const PORT = process.env.PORT;
```

These variables are never reassigned.

---

# When to Use let

Use `let` only when the variable needs to change.

Examples

```js
let count = 0;

count++;
```

```js
let updateData = {};

updateData.title = req.body.title;
updateData.description = req.body.description;
```

The object is built step by step, so `let` is appropriate.

---

# Real Project Usage

### Example 1

```js
const user = await User.findById(id);
```

The variable always refers to the fetched user.

It is never reassigned.

---

### Example 2

```js
const blog = await Blog.findById(blogId);
```

The blog reference never changes.

---

### Example 3

```js
const userId = req.PrcUser._id;
```

The logged-in user's ID remains the same during the request.

---

### Example 4

```js
let updateData = {};
```

Fields are added one by one.

```js
updateData.title = req.body.title;
updateData.description = req.body.description;
```

---

# Why Backend Developers Prefer const

Using `const` prevents accidental reassignment.

Example

```js
const user = await User.findById(id);

// Later in the code

user = null; // ❌ Error
```

JavaScript immediately reports an error.

This helps prevent bugs.

---

# Common Mistakes

### Mistake 1

Using `let` for every variable.

```js
let user = await User.findById(id);
```

If the variable never changes, use `const`.

---

### Mistake 2

Thinking `const` makes objects immutable.

```js
const user = {
    name: "Kirat"
};

user.name = "Singh"; // ✅ Valid
```

Only the reference is constant.

---

### Mistake 3

Trying to reassign a `const` variable.

```js
const PORT = 3000;

PORT = 5000; // ❌ Error
```

---

# Best Practices

- Use `const` by default.
- Use `let` only when reassignment is required.
- Never use `var` in modern backend development.
- Use meaningful variable names.
- Keep variable scope as small as possible.

---

# Summary

- Variables store data.
- `const` is the default choice in backend development.
- `let` is used only when a value needs to change.
- `const` prevents reassignment but does not make objects immutable.
- Choosing `const` helps write safer and more predictable code.
