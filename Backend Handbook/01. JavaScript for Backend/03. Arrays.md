# Arrays

---

# Why

Arrays are used to store multiple values in a single variable.

Instead of creating many separate variables, an array allows us to keep related data together.

Backend applications frequently use arrays when working with:

- Database records
- API responses
- User lists
- Product lists
- Comments
- Categories
- Search results

---

# What

An array is an ordered collection of values.

Each value is stored at a specific position called an **index**.

Syntax

```js
const fruits = ["Apple", "Banana", "Mango"];
```

---

# Array Structure

```js
const array = [
    value1,
    value2,
    value3
];
```

Example

```js
const numbers = [10, 20, 30, 40];
```

---

# Index

Arrays use **zero-based indexing**.

```text
Index

0   1   2

↓

["Apple","Banana","Mango"]
```

---

# Accessing Elements

```js
const fruits = ["Apple", "Banana", "Mango"];

console.log(fruits[0]);
```

Output

```text
Apple
```

---

```js
console.log(fruits[2]);
```

Output

```text
Mango
```

---

# Updating Elements

```js
const fruits = ["Apple", "Banana"];

fruits[1] = "Orange";
```

Result

```js
["Apple", "Orange"]
```

---

# Adding Elements

### push()

Adds an element at the end.

```js
const fruits = ["Apple"];

fruits.push("Banana");
```

Result

```js
["Apple", "Banana"]
```

---

### unshift()

Adds an element at the beginning.

```js
fruits.unshift("Mango");
```

Result

```js
["Mango", "Apple", "Banana"]
```

---

# Removing Elements

### pop()

Removes the last element.

```js
fruits.pop();
```

---

### shift()

Removes the first element.

```js
fruits.shift();
```

---

# Array Length

```js
const fruits = ["Apple", "Banana", "Mango"];

console.log(fruits.length);
```

Output

```text
3
```

# Arrays in Backend

Arrays are very common in backend development.

Database Query

```js
const users = await User.find();
```

Result

```js
[
    { username: "Kirat" },
    { username: "Rahul" },
    { username: "Aman" }
]
```

---

API Response

```js
return res.json({
    data: blogs
});
```

`blogs` is usually an array.

---

Comments

```js
[
    {
        comment: "Nice Post"
    },
    {
        comment: "Very Helpful"
    }
]
```

---

# Common Array Methods

## includes()

Checks whether an element exists.

```js
const roles = ["Admin", "User"];

roles.includes("Admin");
```

Output

```text
true
```

---

## find()

Returns the first matching element.

```js
const users = [
    { id: 1 },
    { id: 2 }
];

const user = users.find(user => user.id === 2);
```

---

## filter()

Returns all matching elements.

```js
const numbers = [10, 20, 30, 40];

const result = numbers.filter(num => num > 20);
```

Result

```js
[30, 40]
```

---

## map()

Creates a new array.

```js
const users = [
    { username: "Kirat" },
    { username: "Rahul" }
];

const names = users.map(user => user.username);
```

Result

```js
["Kirat", "Rahul"]
```

# Common Mistakes

### Mistake 1

Trying to access an invalid index.

```js
const fruits = ["Apple"];

console.log(fruits[5]);
```

Output

```text
undefined
```

---

### Mistake 2

Forgetting that array index starts from **0**.

Wrong

```text
Apple = Index 1 ❌
```

Correct

```text
Apple = Index 0 ✅
```

---

### Mistake 3

Confusing arrays and objects.

Array

```js
["Apple", "Banana"]
```

Object

```js
{
    fruit: "Apple"
}
```

---

# Best Practices

- Store related values together.
- Use meaningful variable names.
- Use array methods instead of writing unnecessary loops.
- Check array length before accessing elements when required.
- Keep arrays organized and avoid unnecessary nesting.

---

# Summary

- Arrays store multiple values.
- Array indexing starts from **0**.
- Arrays are heavily used for database results and API responses.
- Common backend methods include `push()`, `pop()`, `includes()`, `find()`, `filter()`, and `map()`.
- Most database queries that return multiple documents return an array.
