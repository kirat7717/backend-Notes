# Modules (import & export)

---

# Why

As backend applications grow, writing all code in a single file becomes difficult to manage.

Modules allow us to split code into multiple files, making the project:

- Organized
- Reusable
- Easier to maintain
- Easier to debug

Modern Node.js applications use **ES Modules** (`import` and `export`) to share code between files.

---

# What

A module is simply a JavaScript file.

Every file can:

- Export code.
- Import code from another file.

Example

```text
userController.js

↓

Exports registerUser()

↓

userRoutes.js

↓

Imports registerUser()
```

---

# export

The `export` keyword makes a variable, function, or object available to other files.

Syntax

```js
export const variableName = value;
```

Example

```js
export const appName = "Blog API";
```

---

Exporting a Function

```js
export const registerUser = (req, res) => {

};
```

---

Exporting Multiple Values

```js
export const loginUser = () => {

};

export const registerUser = () => {

};
```

Both functions can be imported in another file.

---

# import

The `import` keyword is used to access exported values from another file.

Syntax

```js
import { variableName } from "./file.js";
```

Example

```js
import { registerUser } from "./controllers/userController.js";
```

---

# Importing Multiple Values

```js
import {
    registerUser,
    loginUser
} from "./controllers/userController.js";
```

---

# Default Export

A file can also export one default value.

Export

```js
const connectDB = () => {

};

export default connectDB;
```

Import

```js
import connectDB from "./db/index.js";
```

Notice that curly braces are **not** used.

---

# Named Export vs Default Export

## Named Export

```js
export const registerUser = () => {

};
```

Import

```js
import { registerUser } from "./userController.js";
```

---

## Default Export

```js
export default connectDB;
```

Import

```js
import connectDB from "./db.js";
```

---

# File Extension

When using ES Modules in Node.js, include the `.js` extension.

Correct

```js
import connectDB from "./db/index.js";
```

Wrong

```js
import connectDB from "./db/index";
```

---

# Enabling ES Modules

To use `import` and `export`, add this to `package.json`.

```json
{
    "type": "module"
}
```

Without this, Node.js expects the older CommonJS syntax.

# Common Mistakes

### Mistake 1

Forgetting `"type": "module"` in `package.json`.

This causes `import` statements to fail.

---

### Mistake 2

Using curly braces with a default export.

Wrong

```js
import { connectDB } from "./db.js";
```

Correct

```js
import connectDB from "./db.js";
```

---

### Mistake 3

Forgetting the `.js` extension.

Wrong

```js
import router from "./routes/userRoutes";
```

Correct

```js
import router from "./routes/userRoutes.js";
```

---

### Mistake 4

Importing a named export without curly braces.

Wrong

```js
import registerUser from "./userController.js";
```

Correct

```js
import { registerUser } from "./userController.js";
```

---

# Best Practices

- Keep one responsibility per file.
- Use meaningful file names.
- Group related functions together.
- Use named exports when exporting multiple values.
- Use default exports when a file has one main responsibility.
- Keep the project organized using folders like `controllers`, `routes`, `models`, and `middlewares`.

---

# Summary

- A module is a JavaScript file.
- `export` makes code available to other files.
- `import` brings exported code into another file.
- Use `"type": "module"` to enable ES Modules in Node.js.
- Modules help organize backend projects into smaller, reusable files.
