# Loops

---

# Why

Loops are used to execute the same block of code multiple times without writing it repeatedly.

In backend development, loops are commonly used when processing:

- Database records
- API responses
- Arrays
- Objects
- Validation
- Data transformation

Without loops, developers would have to write the same code again and again.

---

# What

A loop repeatedly executes a block of code until a condition becomes false.

JavaScript provides different types of loops, but in backend development, the most commonly used are:

- `for`
- `for...of`
- `for...in`

---

# for Loop

Used when the number of iterations is known.

Syntax

```js
for (initialization; condition; increment) {

}
```

Example

```js
for (let i = 0; i < 5; i++) {
    console.log(i);
}
```

Output

```text
0
1
2
3
4
```

---

# for...of Loop

Used to iterate over iterable data such as:

- Arrays
- Strings
- Maps
- Sets

Syntax

```js
for (const item of array) {

}
```

Example

```js
const fruits = ["Apple", "Banana", "Mango"];

for (const fruit of fruits) {
    console.log(fruit);
}
```

Output

```text
Apple
Banana
Mango
```

Each iteration gives the **value** of the array element.

---

# for...in Loop

Used to iterate over the **keys** of an object.

Syntax

```js
for (const key in object) {

}
```

Example

```js
const user = {
    username: "Kirat",
    age: 21
};

for (const key in user) {
    console.log(key);
}
```

Output

```text
username
age
```

To access the value:

```js
console.log(user[key]);
```

---

# Difference Between for...of and for...in

| for...of | for...in |
|-----------|-----------|
| Iterates over values | Iterates over keys |
| Used with arrays | Used with objects |
| Returns actual value | Returns property name |

Example

Array

```js
const numbers = [10, 20, 30];

for (const num of numbers) {
    console.log(num);
}
```

Object

```js
const user = {
    username: "Kirat",
    age: 21
};

for (const key in user) {
    console.log(key, user[key]);
}
```

---

# Nested Loops

A loop can exist inside another loop.

Example

```js
const users = [
    {
        username: "Kirat",
        hobbies: ["Coding", "Music"]
    }
];

for (const user of users) {

    for (const hobby of user.hobbies) {
        console.log(hobby);
    }

}
```

# Loops in Backend

Loops are used frequently while processing API data.

Example

```js
const users = await User.find();

for (const user of users) {
    console.log(user.username);
}
```

---

Creating a new array

```js
const usernames = [];

for (const user of users) {
    usernames.push(user.username);
}
```

---

Processing request data

```js
const fields = ["title", "description"];

for (const field of fields) {

}
```

# Real Project Usage

### Example 1

Creating update data dynamically.

```js
const allowedFields = ["title", "description"];

let updateData = {};

for (const field of allowedFields) {

    if (field in req.body) {
        updateData[field] = req.body[field];
    }

}
```

This ensures only allowed fields are updated.

---

### Example 2

Reading object properties.

```js
for (const key in req.body) {
    console.log(key);
}
```

---

### Example 3

Looping through database results.

```js
const blogs = await Blog.find();

for (const blog of blogs) {
    console.log(blog.title);
}
```

# Choosing the Right Loop

Use **for**

- When you need the index.
- When you know the number of iterations.

---

Use **for...of**

- Arrays
- Database results
- Lists

---

Use **for...in**

- Objects
- Dynamic object keys

---

# Common Mistakes

### Mistake 1

Using `for...of` with objects.

```js
const user = {
    username: "Kirat"
};

for (const item of user) {

}
```

❌ Error

Objects are not iterable.

---

### Mistake 2

Using `for...in` for arrays.

```js
const numbers = [10, 20, 30];

for (const index in numbers) {
    console.log(index);
}
```

Output

```text
0
1
2
```

It returns indexes, not values.

---

### Mistake 3

Using dot notation with dynamic keys.

Wrong

```js
for (const key in user) {

    console.log(user.key);

}
```

Correct

```js
for (const key in user) {

    console.log(user[key]);

}
```

---

# Best Practices

- Use `for...of` for arrays.
- Use `for...in` for objects.
- Keep loop bodies simple and readable.
- Avoid deeply nested loops unless necessary.
- Use descriptive variable names.

---

# Summary

- Loops execute code repeatedly.
- `for` is useful when working with indexes.
- `for...of` is used for arrays and iterable values.
- `for...in` is used for object keys.
- Backend developers frequently use loops to process database results, request data, and API responses.
