# Async JavaScript

---

# Why

JavaScript is a **single-threaded** language, meaning it executes one task at a time.

However, backend applications often perform tasks that take time, such as:

- Connecting to a database
- Reading files
- Uploading images
- Sending emails
- Making API requests

If JavaScript waited for each task to finish before doing anything else, the server would become slow and unresponsive.

To solve this problem, JavaScript uses **Asynchronous Programming**.

---

# What

Asynchronous programming allows JavaScript to start a time-consuming task and continue executing other code while waiting for the result.

When the task finishes, JavaScript processes the result.

This makes backend applications faster and more efficient.

---

# Synchronous vs Asynchronous

## Synchronous

Tasks execute one after another.

```text
Task 1

↓

Task 2

↓

Task 3
```

If Task 2 takes a long time, Task 3 must wait.

---

## Asynchronous

JavaScript starts the task and continues with other work.

```text
Start Database Query

↓

Continue Executing Code

↓

Database Responds

↓

Handle Result
```

# Promises

A Promise represents the result of an asynchronous operation.

A Promise has three states:

- Pending
- Fulfilled
- Rejected

Example

```js
const promise = fetch("/users");
```

---

# async Function

The `async` keyword tells JavaScript that the function contains asynchronous code.

Syntax

```js
async function getUser() {

}
```

Arrow Function

```js
const getUser = async () => {

};
```

---

# await

The `await` keyword pauses the execution of the current async function until the Promise is resolved.

Syntax

```js
const result = await promise;
```

Example

```js
const user = await User.findById(id);
```

JavaScript waits for the database to return the user before moving to the next line.

---

# async and await Together

Example

```js
const getUser = async () => {

    const user = await User.findById(id);

    return user;

};
```

---

# Error Handling

Asynchronous operations can fail.

For example:

- Database connection fails
- Invalid ObjectId
- Network error
- File upload fails

Use `try...catch` to handle these errors.

Example

```js
try {

    const user = await User.findById(id);

    return user;

} catch (error) {

    console.log(error);

}
```

# Common Mistakes

### Mistake 1

Using `await` outside an async function.

Wrong

```js
const user = await User.find();
```

Correct

```js
const getUsers = async () => {

    const user = await User.find();

};
```

---

### Mistake 2

Forgetting `await`.

Wrong

```js
const users = User.find();
```

This returns a Promise instead of the actual data.

Correct

```js
const users = await User.find();
```

---

### Mistake 3

Ignoring error handling.

Wrong

```js
const user = await User.findById(id);
```

Better

```js
try {

    const user = await User.findById(id);

} catch (error) {

    console.log(error);

}
```

---

### Mistake 4

Using asynchronous code unnecessarily.

Not every function needs to be async.

Use `async` only when working with Promises.

---

# Best Practices

- Use `async/await` instead of long Promise chains whenever possible.
- Always use `try...catch` when handling asynchronous operations.
- Await database queries before using their results.
- Keep async functions small and focused.
- Return meaningful errors when asynchronous operations fail.

---

# Summary

- Backend applications frequently perform asynchronous tasks.
- `async` creates an asynchronous function.
- `await` waits for a Promise to resolve.
- Database operations in Mongoose return Promises and are commonly used with `await`.
- Use `try...catch` to handle errors during asynchronous operations.
- `async/await` makes asynchronous code easier to read and maintain than traditional Promise chains.
