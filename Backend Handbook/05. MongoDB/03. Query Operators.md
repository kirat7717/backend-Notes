# Query Operators

---

# Why

Sometimes we don't want all documents from the database.

Instead, we want to:

- Find users older than 18
- Find blogs created by a specific user
- Find products costing less than ₹500
- Search documents with multiple conditions

Query Operators help us perform these advanced searches.

---

# What are Query Operators?

Query Operators are special keywords provided by MongoDB that allow us to filter documents based on specific conditions.

Most MongoDB operators begin with the **`$`** symbol.

Example

```js
{
    age: {
        $gt: 18
    }
}
```

---

# Equality Operator

Find documents with an exact value.

Example

```js
const user = await User.find({
    username: "Kirat"
});
```

Equivalent Query

```text
username = "Kirat"
```

---

# $gt (Greater Than)

Returns documents where the value is greater than the specified value.

Example

```js
const users = await User.find({
    age: {
        $gt: 18
    }
});
```

---

# $gte (Greater Than or Equal)

Example

```js
const users = await User.find({
    age: {
        $gte: 18
    }
});
```

---

# $lt (Less Than)

Example

```js
const users = await User.find({
    age: {
        $lt: 30
    }
});
```

---

# $lte (Less Than or Equal)

Example

```js
const users = await User.find({
    age: {
        $lte: 30
    }
});
```

---

# $ne (Not Equal)

Returns documents whose value is not equal to the specified value.

Example

```js
const users = await User.find({
    username: {
        $ne: "Kirat"
    }
});
```

---

# $in

Matches any value inside an array.

Example

```js
const users = await User.find({
    role: {
        $in: ["admin", "editor"]
    }
});
```

---

# $nin

Matches values **not** inside an array.

Example

```js
const users = await User.find({
    role: {
        $nin: ["admin", "editor"]
    }
});
```

---

# $and

All conditions must be true.

Example

```js
const users = await User.find({
    $and: [
        { age: { $gte: 18 } },
        { city: "Bathinda" }
    ]
});
```

---

# $or

At least one condition must be true.

Example

```js
const users = await User.find({
    $or: [
        { city: "Bathinda" },
        { city: "Chandigarh" }
    ]
});
```

---

# $exists

Checks whether a field exists.

Example

```js
const users = await User.find({
    phone: {
        $exists: true
    }
});
```

# Update Operators

MongoDB also provides operators for updating documents.

---

## $set

Updates specific fields.

Example

```js
await Blog.updateOne(
    {
        _id: blogId
    },
    {
        $set: {
            title: "Updated Title"
        }
    }
);
```

---

## $inc

Increases a numeric value.

Example

```js
await User.updateOne(
    {
        _id: id
    },
    {
        $inc: {
            loginCount: 1
        }
    }
);
```

---

## $push

Adds a value to an array.

Example

```js
await User.updateOne(
    {
        _id: id
    },
    {
        $push: {
            hobbies: "Coding"
        }
    }
);
```

---

## $pull

Removes a value from an array.

Example

```js
await User.updateOne(
    {
        _id: id
    },
    {
        $pull: {
            hobbies: "Coding"
        }
    }
);
```

---

# Sorting

Sort results in ascending or descending order.

Newest First

```js
const blogs = await Blog.find()
    .sort({
        createdAt: -1
    });
```

Oldest First

```js
const blogs = await Blog.find()
    .sort({
        createdAt: 1
    });
```

---

# Limiting Results

Return only a fixed number of documents.

Example

```js
const blogs = await Blog.find()
    .limit(10);
```

---

# Skipping Results

Useful for pagination.

Example

```js
const blogs = await Blog.find()
    .skip(10)
    .limit(10);
```

---

# Combining Queries

Example

```js
const blogs = await Blog.find({
    authorId: userId
})
.sort({
    createdAt: -1
})
.limit(5);
```

# Common Mistakes

### Mistake 1

Using `$set` when replacing an entire document.

`$set` updates only specified fields.

---

### Mistake 2

Forgetting comparison operators.

Wrong

```js
age: 18
```

Correct

```js
age: {
    $gte: 18
}
```

---

### Mistake 3

Using `find()` when expecting one document.

Prefer

```js
findOne()
```

---

### Mistake 4

Forgetting `await`.

Wrong

```js
const users = User.find();
```

Correct

```js
const users = await User.find();
```

---

# Best Practices

- Use `findOne()` when expecting one result.
- Use comparison operators for filtering.
- Use `$set` for partial updates.
- Use `.sort()` for ordering results.
- Use `.limit()` and `.skip()` for pagination.
- Combine operators to write efficient queries.

---

# Summary

- Query Operators filter documents based on conditions.
- Common operators include `$gt`, `$gte`, `$lt`, `$lte`, `$ne`, `$in`, `$or`, and `$and`.
- Update operators like `$set`, `$inc`, `$push`, and `$pull` modify documents.
- `.sort()`, `.limit()`, and `.skip()` help organize and paginate query results.
- Mastering query operators is essential for building powerful and efficient MongoDB applications.
