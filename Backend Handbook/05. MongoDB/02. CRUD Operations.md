# CRUD Operations

---

# Why

A database is useful only if we can work with its data.

In almost every application, we need to:

- Create new data
- Read existing data
- Update existing data
- Delete unwanted data

These four basic operations are known as **CRUD Operations**.

---

# What is CRUD?

CRUD is an acronym for:

- **C** → Create
- **R** → Read
- **U** → Update
- **D** → Delete

Every backend application performs these operations on the database.

---

---

# CREATE

Create adds a new document to a collection.

Example

```js
await User.create({
    username: "Kirat",
    email: "kirat@gmail.com"
});
```

Returns

```text
Newly Created Document
```

---

# READ

Read retrieves documents from the database.

Read All Documents

```js
const users = await User.find();
```

Read One Document

```js
const user = await User.findOne({
    email: "kirat@gmail.com"
});
```

Read by ID

```js
const user = await User.findById(id);
```

---

# UPDATE

Update modifies existing documents.

Update One

```js
await Blog.updateOne(
    {
        _id: blogId
    },
    {
        $set: {
            title: "Updated Title"
        }
    }
);
```

---

Find and Update

```js
const blog = await Blog.findOneAndUpdate(
    {
        _id: blogId
    },
    {
        $set: {
            title: "Updated Title"
        }
    },
    {
        new: true
    }
);
```

Returns the updated document.

---

# DELETE

Delete removes documents.

Delete by ID

```js
await Blog.findByIdAndDelete(blogId);
```

Delete One

```js
await Blog.deleteOne({
    _id: blogId
});
```

# CRUD with HTTP Methods

| CRUD | HTTP Method |
|------|-------------|
| Create | POST |
| Read | GET |
| Update | PATCH / PUT |
| Delete | DELETE |

|---------|---------|

# Common Mistakes

### Mistake 1

Using `find()` when expecting one document.

Wrong

```js
const user = await User.find({
    email
});
```

Returns

```text
Array
```

Correct

```js
const user = await User.findOne({
    email
});
```

---

### Mistake 2

Forgetting `await`.

Wrong

```js
const blogs = Blog.find();
```

Returns a Promise.

Correct

```js
const blogs = await Blog.find();
```

---

### Mistake 3

Using `updateOne()` when updated data is needed.

`updateOne()` returns update information.

If you need the updated document, use

```js
findOneAndUpdate()
```

with

```js
new: true
```

---

### Mistake 4

Updating everything from `req.body`.

Always allow only specific fields.

Example

```js
const allowedFields = [
    "title",
    "description"
];
```

---

# Best Practices

- Use `findOne()` when only one document is expected.
- Use `findById()` when searching by `_id`.
- Use `findOneAndUpdate()` when returning updated data.
- Validate request data before CRUD operations.
- Return appropriate HTTP status codes.
- Restrict which fields users can update.

---

# Summary

- CRUD stands for Create, Read, Update, and Delete.
- CRUD operations are the foundation of every backend application.
- Mongoose provides methods such as `create()`, `find()`, `findOne()`, `findById()`, `findOneAndUpdate()`, and `findByIdAndDelete()`.
- Choosing the correct CRUD method improves code readability and application performance.
