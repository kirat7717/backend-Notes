# MongoDB Basics

---

# Why

Every application needs a place to store data permanently.

Examples:

- Users
- Blogs
- Comments
- Products
- Orders

Without a database, all data would be lost whenever the server restarts.

MongoDB provides a fast and flexible way to store and retrieve data.

---

# What is MongoDB?

MongoDB is a **NoSQL Document Database**.

Instead of storing data in rows and columns like traditional SQL databases, MongoDB stores data as **documents**.

These documents are stored inside **collections**.

---

# MongoDB Hierarchy

```text
Database

↓

Collection

↓

↓

Fields
```

Example

```text
BlogDB

↓

blogs

↓

{
    title: "Learning MongoDB",
    author: "Kirat"
}

↓

title

```

---

# Database

A database is the highest level container.

It stores multiple collections.

Example

```text
BlogDB

UserDB

SchoolDB
```

---

# Collection

A collection is similar to a table in SQL.

It stores related documents.

Example

```text
users

blogs

comments

categories
```

---

# Document

A document is a single record inside a collection.

Documents are stored in **BSON (Binary JSON)** format.

Example

```json
{
    "_id": "...",
    "title": "Learning MongoDB",
    "description": "MongoDB Notes",
    "author": "Kirat"
}
```

---

# Fields

A document contains fields.

Example

```json
{
    "username": "Kirat",
    "email": "kirat@gmail.com",
    "age": 21
}
```

Fields are similar to columns in SQL.

---

# BSON

MongoDB stores data internally as **BSON (Binary JSON)**.

BSON supports additional data types such as:

- ObjectId
- Date
- Boolean
- Number
- Array
- Object

Although MongoDB stores BSON internally, developers usually work with JSON-like objects.

---

# ObjectId

Every MongoDB document automatically receives a unique `_id`.

Example

```json
{
    "_id": "687a7c5e2ab9a...",
    "title": "Node.js"
}
```

Purpose:

- Uniquely identifies each document.
- Used to find, update, and delete documents.

---

# Common Misconceptions

### Mistake 1

Thinking MongoDB stores tables.

❌ Wrong

MongoDB stores **collections**.

---

### Mistake 2

Thinking documents are rows.

A document is similar to a row, but it is actually a complete JSON-like object.

---

### Mistake 3

Thinking every document has the same fields.

MongoDB allows flexible document structures, although in real projects Mongoose schemas enforce consistency.

---

# Best Practices

- Create meaningful collection names.
- Store related data together.
- Let MongoDB generate `_id`.
- Use Mongoose schemas for consistency.
- Design collections based on application requirements.

---

# Summary

- MongoDB is a NoSQL document database.
- Data is stored as documents inside collections.
- Collections are stored inside databases.
- Every document has a unique `_id`.
- MongoDB stores data internally as BSON.
- MongoDB is the database commonly used in the MERN stack.
