# Response Object (`res`)

---

# Why

After processing a client's request, the server must send a response back.

The response tells the client:

- Whether the request was successful.
- What data was returned.
- What error occurred (if any).

Express provides the **Response Object (`res`)** for this purpose.

Without `res`, the client would never receive a reply from the server.

---

# What

The Response Object (`res`) is an object provided by Express that is used to send a response back to the client.

Every route handler receives it as the second parameter.

Example

```js
app.get("/users", (req, res) => {

});
```

Here,

- `req` → Request Object
- `res` → Response Object

---

# Common Response Methods

| Method | Purpose |
|---------|---------|
| `res.status()` | Sets the HTTP status code |
| `res.json()` | Sends JSON data |
| `res.send()` | Sends any type of response |
| `res.redirect()` | Redirects to another URL |
| `res.download()` | Sends a file for download |
| `res.sendFile()` | Sends a file |

In backend API development, the most commonly used methods are:

- `res.status()`
- `res.json()`

---

# res.status()

Sets the HTTP status code.

Syntax

```js
res.status(statusCode)
```

Example

```js
res.status(200);
```

---

# res.json()

Sends a JSON response to the client.

Syntax

```js
res.json(data)
```

Example

```js
res.json({
    message: "Success"
});
```

---

# Using status() and json() Together

This is the most common way to send API responses.

Example

```js
return res.status(200).json({
    message: "Success",
    data: user
});
```

Flow

```text
Server

↓

Status Code

↓

JSON Data

↓

Client
```

---

# res.send()

Sends a response of any type.

Example

```js
res.send("Server Running");
```

or

```js
res.send({
    message: "Success"
});
```

For REST APIs, `res.json()` is generally preferred because APIs usually return JSON.

---

# res.redirect()

Redirects the client to another route.

Example

```js
res.redirect("/login");
```

---

# res.download()

Sends a file as a download.

Example

```js
res.download("./files/report.pdf");
```

---

# res.sendFile()

Sends a file to the client.

Example

```js
res.sendFile("index.html");
```

---

# JSON Response Structure

A good API response should be consistent.

Example

```js
return res.status(200).json({
    message: "Blogs fetched successfully",
    data: blogs
});
```

Error Response

```js
return res.status(404).json({
    message: "Blog not found"
});
```

---


# Common Mistakes

### Mistake 1

Sending a response without a status code.

```js
res.json({
    message: "Success"
});
```

Although Express defaults to `200`, it's better to specify the status explicitly.

Correct

```js
res.status(200).json({
    message: "Success"
});
```

---

### Mistake 2

Sending multiple responses.

Wrong

```js
res.status(200).json({
    message: "Success"
});

res.status(400).json({
    message: "Error"
});
```

A request should receive only **one** response.

---

### Mistake 3

Forgetting to use `return`.

Wrong

```js
if (!user) {

    res.status(404).json({
        message: "User not found"
    });

}

console.log("This code still executes.");
```

Better

```js
if (!user) {

    return res.status(404).json({
        message: "User not found"
    });

}
```

---

# Best Practices

- Always return meaningful HTTP status codes.
- Keep response structure consistent.
- Use `res.json()` for REST APIs.
- Return immediately after sending a response.
- Send only one response per request.

---

# Summary

- The Response Object (`res`) is used to send data back to the client.
- `res.status()` sets the HTTP status code.
- `res.json()` sends JSON responses.
- Most REST APIs use `res.status().json()`.
- Every request should receive only one response.
- Consistent API responses improve readability and maintainability.
