# HTTP Methods

---

# Why

A client and a server communicate using **HTTP Requests**.

Every request has a purpose.

For example:

- Fetch user data
- Create a new blog
- Update a blog
- Delete a blog

Instead of using one request type for everything, HTTP provides different methods for different operations.

This makes APIs clear, predictable, and easier to understand.

---

# What

An HTTP Method tells the server **what action the client wants to perform**.

Each method has a specific purpose.

The most commonly used methods are:

- GET
- POST
- PUT
- PATCH
- DELETE

---

# GET

Used to retrieve data from the server.

It should **never modify data**.

Syntax

```http
GET /blogs
```

Example

```http
GET /users
```

Purpose

- Fetch all users
- Fetch a single blog
- Fetch profile information

---

# POST

Used to create new data.

Syntax

```http
POST /blogs
```

Example

```http
POST /register
```

Purpose

- Register user
- Login user
- Create blog
- Upload file

---

# PUT

Used to replace an entire resource.

The client sends all fields.

Example

```http
PUT /blogs/123
```

Suppose a blog contains

```text
Title
Description
Category
```

Using PUT means sending all fields again, even if only one field changed.

---

# PATCH

Used to update only specific fields.

Only changed fields are sent.

Example

```http
PATCH /blogs/123
```

Request Body

```json
{
    "title": "Updated Title"
}
```

Only the title is updated.

---

# DELETE

Used to remove data.

Syntax

```http
DELETE /blogs/123
```

Purpose

- Delete blog
- Delete user
- Delete comment

---

# HTTP Methods in CRUD

| CRUD Operation | HTTP Method |
|---------------|-------------|
| Create | POST |
| Read | GET |
| Update | PUT / PATCH |
| Delete | DELETE |

---

# PUT vs PATCH

| PUT | PATCH |
|------|-------|
| Replaces the entire resource | Updates only selected fields |
| Sends all fields | Sends only changed fields |
| Less common in modern APIs | Most commonly used |

# Real Project Usage

### Register User

```http
POST /api/v1/register
```

Creates a new user.

---

### Login User

```http
POST /api/v1/login
```

Authenticates a user.

---

### Get All Blogs

```http
GET /api/v1/blogs
```

Returns all blogs.

---

### Get Single Blog

```http
GET /api/v1/blogs/:id
```

Returns one blog.

---

### Update Blog

```http
PATCH /api/v1/blogs/:id
```

Updates only the fields sent by the client.

---

### Delete Blog

```http
DELETE /api/v1/blogs/:id
```

Deletes the selected blog.

---

# Common Mistakes

### Mistake 1

Using GET to create data.

Wrong

```http
GET /createBlog
```

Correct

```http
POST /blogs
```

---

### Mistake 2

Using POST for updates.

Wrong

```http
POST /blogs/123
```

Correct

```http
PATCH /blogs/123
```

---

### Mistake 3

Using PUT when only one field changes.

If only one field needs updating, prefer PATCH.

---

# Best Practices

- Use GET only for fetching data.
- Use POST only for creating new resources.
- Use PATCH for partial updates.
- Use DELETE only for removing resources.
- Choose HTTP methods based on their intended purpose.

---

# Summary

- HTTP methods define the action a client wants to perform.
- GET retrieves data.
- POST creates new data.
- PUT replaces an entire resource.
- PATCH updates selected fields.
- DELETE removes data.
- Following standard HTTP methods makes APIs easier to understand and maintain.
