# Request Object (`req`)

---

# Why

When a client sends an HTTP request to the server, the server needs access to information such as:

- Request body
- URL parameters
- Query parameters
- Headers
- Logged-in user information

Express provides the **Request Object (`req`)** to access all of this data.

Without `req`, the server would not know what the client is asking for.

---

# What

The Request Object (`req`) is an object created by Express for every incoming HTTP request.

It contains all the information sent by the client.

Example

```js
app.get("/users", (req, res) => {

});
```

Here,

- `req` â†’ Request Object
- `res` â†’ Response Object

---

# Common Properties of Request Object

| Property | Purpose |
|----------|---------|
| `req.body` | Request body data |
| `req.params` | Route parameters |
| `req.query` | Query parameters |
| `req.headers` | Request headers |
| `req.cookies` | Cookies sent by client |
| `req.method` | HTTP method |
| `req.url` | Requested URL |
| `req.originalUrl` | Original request URL |

---

# req.body

Contains data sent in the request body.

Usually used with:

- POST
- PUT
- PATCH

Example Request

```json
{
    "title": "Learning Express",
    "description": "Request Object"
}
```

Access

```js
const { title, description } = req.body;
```

---

# req.params

Contains dynamic values from the URL.

Route

```js
router.get("/blogs/:id", getBlog);
```

Request

```text
GET /blogs/123
```

Access

```js
const { id } = req.params;
```

Result

```text
123
```

---

# req.query

Contains query parameters.

Request

```text
GET /blogs?page=1&limit=10
```

Access

```js
const { page, limit } = req.query;
```

Result

```text
page = 1
limit = 10
```

---

# req.headers

Contains request headers sent by the client.

Example

```js
const token = req.headers.authorization;
```

Commonly used for:

- JWT Authentication
- Content-Type
- Authorization

---

# req.method

Returns the HTTP method.

Example

```js
console.log(req.method);
```

Output

```text
GET
```

or

```text
POST
```

---

# req.url

Returns the requested URL.

Example

```js
console.log(req.url);
```

Output

```text
/blogs/123
```

---

# Custom Properties

Middleware can add custom properties to the Request Object.

Example

```js
req.PrcUser = user;
```

Now every controller can access:

```js
const userId = req.PrcUser._id;
```

This is commonly used after JWT authentication.

# Real Project Usage

### Reading Request Body

```js

```

```js

```

---

---


---

```js

```

# Common Mistakes

### Mistake 1

Trying to access `req.body` without JSON middleware.

Wrong

```js
app.post("/register", (req, res) => {

    console.log(req.body);

});
```

If `express.json()` is not used,

```js
req.body
```

will be

```text
undefined
```

---

### Mistake 2

Using the wrong property.

Wrong

```js
req.body.id
```

when the ID is actually in

```js
req.params.id
```

---

### Mistake 3

Confusing `req.params` and `req.query`.

```text
/blogs/123

â†“

req.params.id
```

```text
/blogs?page=1

â†“

req.query.page
```

# Best Practices

- Use `req.body` for request data.
- Use `req.params` for dynamic URL values.
- Use `req.query` for filtering and pagination.
- Store authenticated user information on the Request Object in middleware.
- Validate all incoming request data before using it.

---

# Summary

- The Request Object (`req`) contains all information sent by the client.
- `req.body` stores request body data.
- `req.params` stores dynamic URL parameters.
- `req.query` stores query parameters.
- `req.headers` stores request headers.
- Middleware can attach custom properties like the authenticated user.
- Every Express controller receives a Request Object for processing client requests.
