# HTTP Request Lifecycle (API Lifecycle)

---

# Why

Whenever a user interacts with an application, a request is sent to the backend.

Understanding the complete journey of a request helps developers:

- Debug errors
- Build better APIs
- Understand Express internally
- Know where middleware, controllers, and databases fit into the application

Every API request follows a lifecycle from the client to the server and back.

---

# What

The HTTP Request Lifecycle is the complete journey of a request from the client to the server and the response back to the client.

Every request follows a sequence of steps before the user receives a response.

---

# Complete Lifecycle

```text
Client

↓

HTTP Request

↓

Express Server

↓

Middlewares

↓

Route Matching

↓

Controller

↓

Business Logic

↓

Database

↓

Database Response

↓

Controller

↓

Response Object (res)

↓

HTTP Response

↓

```

---

# Step 1 — Client Sends Request

The lifecycle begins when the client sends an HTTP request.

The client can be:

- Browser
- React Application
- Mobile App
- Postman

Example

```http
GET /api/v1/blogs
```

---

# Step 2 — Request Reaches Express Server

The request reaches the Express application.

Example

```js
const app = express();
```

Express receives the request and starts processing it.

---

# Step 3 — Middleware Executes

Before reaching the route, middleware runs.

Examples

- JSON Parser
- Authentication
- Logging
- Validation

Example

```js
app.use(express.json());
```

If authentication middleware exists:

```js
app.use(authentication);
```

The middleware decides whether the request should continue.

---

# Step 4 — Route Matching

Express checks whether a matching route exists.

Example

```js
router.get("/blogs", getBlogs);
```

If no matching route is found,

the server returns

```text
404 Not Found
```

---

# Step 5 — Controller Executes

The controller contains the business logic.

Example

```js
export const getBlogs = async (req, res) => {

};
```

Here, the application decides what should happen.

---

# Step 6 — Business Logic

The controller performs operations such as:

- Validation
- Authentication checks
- Authorization
- Processing request data

Example

```js
if (!title) {

    return res.status(400).json({
        message: "Title is required"
    });

}
```

---

# Step 7 — Database Operation

If data is required, the controller communicates with the database.

Examples

```js
await User.find();
```

```js
await Blog.findById(id);
```

```js
await Blog.create(data);
```

The database processes the query and returns the result.

---

# Step 8 — Database Response

MongoDB returns the requested data.

Example

```js
const blogs = await Blog.find();
```

Now the controller has the data.

---

# Step 9 — Prepare Response

The controller prepares the response.

Example

```js
return res.status(200).json({
    message: "Blogs fetched successfully",
    data: blogs
});
```

---

# Step 10 — Response Sent to Client

Express sends the HTTP response.

The client receives:

- Status Code
- JSON Data
- Headers

Example

```json
{
    "message": "Blogs fetched successfully",
    "data": []
}
```

The lifecycle is now complete.

# Where Each Component Fits

| Component | Responsibility |
|-----------|----------------|
| Client | Sends request |
| Express | Receives request |
| Middleware | Pre-processes request |
| Route | Finds matching endpoint |
| Controller | Executes business logic |
| Database | Stores and retrieves data |
| Response Object | Sends response |
| Client | Displays result |

---

# Common Mistakes

### Mistake 1

Thinking the request goes directly to the controller.

In reality, middleware executes before the controller.

---

### Mistake 2

Thinking the database sends responses.

The database only returns data.

The controller prepares the HTTP response.

---

### Mistake 3

Putting business logic inside routes.

Routes should only connect endpoints to controllers.

Business logic belongs in controllers.

---

# Best Practices

- Keep routes simple.
- Use middleware for common tasks.
- Keep controllers focused on business logic.
- Validate data before accessing the database.
- Return consistent responses.
- Keep database operations inside controllers or services.

---

# Summary

- Every API request follows a lifecycle from the client to the server and back.
- The request passes through middleware, routes, controllers, and the database before a response is sent.
- Understanding the request lifecycle helps in debugging, designing APIs, and building scalable backend applications.
- A clear separation of responsibilities makes backend applications easier to maintain and extend.
