# Error Handling

---

# Why

Errors are a normal part of backend development.

Examples include:

- Invalid user input
- Database connection failure
- Missing JWT token
- Blog not found
- Internal server errors

A backend application should handle these errors properly instead of crashing.

Error handling allows us to send meaningful responses to the client.

---

# What is Error Handling?

Error Handling is the process of detecting, managing, and responding to errors that occur during application execution.

Instead of letting the application crash, we catch errors and send an appropriate response.

---

# Types of Errors

### Client Errors (4xx)

Caused by incorrect requests from the client.

Examples

- Missing fields
- Invalid data
- Unauthorized user
- Resource not found

---

### Server Errors (5xx)

Caused by problems on the server.

Examples

- Database connection failure
- Unexpected exceptions
- Server crashes

# try...catch

JavaScript uses `try...catch` to handle runtime errors.

Syntax

```js
try {

    // Code

} catch (error) {

    // Handle Error

}
```

---

# Example

```js
try {

    const blogs = await Blog.find();

    return res.status(200).json({
        data: blogs
    });

} catch (error) {

    return res.status(500).json({
        message: error.message
    });

}
```

---

    message: "All fields are required"

---

# Error Handling in Controllers

Example

```js
export const getBlogs = async (req, res) => {

    try {

        const blogs = await Blog.find();

        return res.status(200).json({
            data: blogs
        });

    } catch (error) {

        return res.status(500).json({
            message: error.message
        });

    }

};
```

# Centralized Error Handling

As projects grow, writing `try...catch` inside every controller becomes repetitive.

A professional approach is to use:

- Async Handler
- Global Error Middleware
- Custom Error Classes

Example

```text
Controller

↓

Throw Error

↓

Global Error Middleware

↓

Response
```

This keeps controllers clean.

# Common Mistakes

### Mistake 1

Ignoring errors.

Wrong

```js
await Blog.find();
```

If an error occurs, the application may crash.

---

### Mistake 2

Returning `200 OK` for failed requests.

Wrong

```js
return res.status(200).json({
    message: "Blog not found"
});
```

Correct

```js
return res.status(404).json({
    message: "Blog not found"
});
```

---

### Mistake 3

Using `500` for validation errors.

Validation errors should return:

```text
400 Bad Request
```

---

### Mistake 4

Exposing sensitive error details.

Wrong

```js
return res.status(500).json({
    error
});
```

Instead, return a meaningful message.

# Best Practices

- Use `try...catch` for asynchronous operations.
- Return appropriate HTTP status codes.
- Send clear and consistent error messages.
- Handle expected errors gracefully.
- Use centralized error handling in large applications.

# Summary

- Error Handling prevents applications from crashing.
- `try...catch` is used to catch runtime errors.
- Always return the correct HTTP status code.
- Client errors use `4xx` status codes.
- Server errors use `5xx` status codes.
- Large applications use centralized error handling to keep controllers clean and maintainable.
