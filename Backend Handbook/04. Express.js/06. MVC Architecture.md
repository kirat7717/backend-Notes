# MVC Architecture

---

# Why

As applications grow, writing everything in one file becomes difficult.

Problems include:

- Large files
- Mixed responsibilities
- Difficult debugging
- Difficult maintenance
- Poor scalability

MVC solves these problems by separating the application into different layers, where each layer has a specific responsibility.

---

# What is MVC?

MVC stands for:

- **M** → Model
- **V** → View
- **C** → Controller

It is a software architecture pattern that separates different parts of an application.

In backend development with Express, we mainly work with:

- Model
- Controller

Since we are building APIs instead of HTML pages, the frontend (React) acts as the View.

---

# MVC Components

## Model

The Model manages the application's data.

Responsibilities:

- Define database schema
- Communicate with the database
- Perform CRUD operations

Example

```js
const blogs = await Blog.find();
```

---

## Controller

The Controller contains the business logic.

Responsibilities:

- Read request data
- Validate input
- Call models
- Send responses

Example

```js
export const getBlogs = async (req, res) => {

};
```

---

## View

The View displays data to the user.

In a MERN application:

```text
React = View
```

React receives JSON from the backend and displays it to users.

# Responsibilities

## Routes

- Receive requests
- Call controllers

Example

```js
router.get("/blogs", getBlogs);
```

---

## Controllers

- Process request
- Apply business logic
- Call models
- Return response

---

## Models

- Communicate with MongoDB
- Store data
- Retrieve data

---

## Middleware

- Authentication
- Validation
- Logging

Runs before controllers.

# Real Project Example

## Route

```js
router.patch(
    "/blogs/:id",
    authentication,
    updateBlog
);
```

---

## Controller

```js
export const updateBlog = async (req, res) => {

    const updatedBlog = await Blog.findOneAndUpdate();

    return res.status(200).json({
        data: updatedBlog
    });

};
```

---

## Model

```js
const updatedBlog = await Blog.findOneAndUpdate();
```

The model communicates with MongoDB.

---

## View

React receives

```json
{
    "data": {
        "title": "Updated Blog"
    }
}
```

and updates the UI.

# Common Mistakes

### Mistake 1

Writing business logic inside routes.

Wrong

```js
router.post("/register", async (req, res) => {

    // 100 lines of code

});
```

Correct

```js
router.post(
    "/register",
    registerUser
);
```

---

### Mistake 2

Putting database queries inside routes.

Database operations belong in controllers (or service layer in larger applications).

---

### Mistake 3

Thinking React is separate from MVC.

In MERN,

```text
React = View
```

The frontend acts as the View layer.

---

### Mistake 4

Giving one file multiple responsibilities.

Keep each layer focused on its own task.

---

# Best Practices

- Keep routes simple.
- Keep controllers focused on business logic.
- Keep models responsible for database operations.
- Use middleware for authentication and validation.
- Follow a modular folder structure.
- Give each component a single responsibility.

---

# Summary

- MVC stands for Model, View, and Controller.
- The Model manages data and communicates with the database.
- The Controller contains business logic and handles requests.
- In MERN, React acts as the View.
- MVC separates responsibilities, making applications easier to maintain, scale, and test.
- Most professional Express applications follow the MVC architecture.
