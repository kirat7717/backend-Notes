# Express Basics

---

# Why

Node.js provides the ability to run JavaScript outside the browser, but it does not provide an easy way to build web servers or APIs.

Writing everything using Node's built-in `http` module becomes difficult as projects grow.

Express.js simplifies backend development by providing a clean and organized way to build web applications and REST APIs.

---

# What

Express.js is a lightweight web framework built on top of Node.js.

It provides features such as:

- Routing
- Middleware
- Request Handling
- Response Handling
- Error Handling

Express allows developers to build backend applications faster with less code.

# Creating an Express Application

```js
import express from "express";

const app = express();
```

- `express()` creates an Express application.
- `app` is the main object used to configure the server.

---

# Creating a Server

```js
const PORT = 3000;

app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});
```

`app.listen()` starts the server and waits for incoming requests.

---

# Creating a Route

A route defines how the server responds to a specific request.

Example

```js
app.get("/", (req, res) => {

    res.send("Server Running");

});
```

When the browser visits

```text
http://localhost:3000/
```

the server responds with:

```text
Server Running
```


Example

```js
app.post("/register", (req, res) => {
```

---

# Parsing JSON

Express cannot read JSON request bodies automatically.

Use:

```js
app.use(express.json());
```

Now requests like

```json
{
    "username": "Kirat"
}
```

can be accessed using

```js
req.body
```

---

# Using Middleware

Middleware runs before the request reaches the route.

Example

```js
app.use(express.json());
```

Every incoming request passes through this middleware first.

# Common Mistakes

### Mistake 1

Forgetting to call `express()`.

Wrong

```js
const app = express;
```

Correct

```js
const app = express();
```

---

### Mistake 2

Forgetting `express.json()`.

Without it,

```js
req.body
```

will be

```text
undefined
```

for JSON requests.

---

### Mistake 3

Hardcoding the port.

Wrong

```js
const PORT = 3000;
```

Better

```js
const PORT = process.env.PORT;
```

---

### Mistake 4

Putting all routes in one file.

As the project grows, separate routes into dedicated files.

---

# Best Practices

- Keep the Express application small and organized.
- Use `express.json()` for JSON APIs.
- Store routes in separate files.
- Store business logic inside controllers.
- Use environment variables for configuration.
- Follow a modular folder structure.

---

# Summary

- Express.js is a web framework built on top of Node.js.
- It simplifies backend development by providing routing, middleware, and request/response handling.
- An Express application is created using `express()`.
- `app.listen()` starts the server.
- `express.json()` parses incoming JSON requests.
- Express forms the foundation of most Node.js backend applications.
