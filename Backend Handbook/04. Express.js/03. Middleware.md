# Middleware

---

# Why

Before a request reaches the controller, we often need to perform some common tasks.

Examples:

- Parse JSON data
- Authenticate the user
- Check permissions
- Log requests
- Validate input

Instead of writing this code inside every controller, Express provides **Middleware**.

Middleware helps keep the application clean, reusable, and organized.

---

# What is Middleware?

Middleware is a function that executes **between receiving the request and sending the response**.

It has access to:

- Request Object (`req`)
- Response Object (`res`)
- Next Middleware (`next`)

General Syntax

```js
(req, res, next)
```

---

# Request Flow

```text
Client

↓

Request

↓

Middleware

↓

Route

↓

Controller

↓

Response

↓

Client
```

A request can pass through multiple middleware functions before reaching the controller.

---

# Middleware Syntax

```js
const middleware = (req, res, next) => {

    // Work

    next();

};
```

---

# What is next()?

`next()` tells Express:

> "I'm done. Continue to the next middleware or controller."

If `next()` is not called and no response is sent, the request will remain pending.

---

# Example

```js
const logger = (req, res, next) => {

    console.log("Request Received");

    next();

};
```

Using it

```js
app.use(logger);
```

Now every request prints

```text
Request Received
```

before reaching the controller.

---

# Built-in Middleware

Express provides several built-in middleware.

### express.json()

Parses incoming JSON requests.

```js
app.use(express.json());
```

Without it,

```js
req.body
```

will be

```text
undefined
```

---

### express.static()

Serves static files.

```js
app.use(express.static("public"));
```

---

# Application-Level Middleware

Runs for every request.

```js
app.use((req, res, next) => {

    console.log("Middleware");

    next();

});
```

---

# Route-Level Middleware

Runs only for a specific route.

```js
router.get(
    "/profile",
    authentication,
    getProfile
);
```

Here,

```text
authentication
```

runs only for

```text
GET /profile
```

---

# Custom Middleware

We can create our own middleware.

Example

```js
const authentication = (req, res, next) => {

    // Verify JWT

    next();

};
```

Store it inside

```text
middlewares/

authentication.js
```

---

# Multiple Middleware

A request can pass through multiple middleware.

```js
router.post(

    "/blogs",

    authentication,

    validateBlog,

    createBlog

);
```

Flow

```text
Request

↓

authentication

↓

validateBlog

↓

createBlog
```

---

# Middleware Can

- Read Request
- Modify Request
- Send Response
- End Request
- Call next()

---

# Adding Data to Request

Middleware can add custom properties.

Example

```js
req.PrcUser = user;

next();
```

Later inside the controller

```js
const userId = req.PrcUser._id;
```

This is how authentication middleware shares the logged-in user.

# Common Mistakes

### Mistake 1

Forgetting `next()`.

Wrong

```js
const middleware = (req, res, next) => {

    console.log("Hello");

};
```

The request never reaches the controller.

---

### Mistake 2

Calling `next()` after sending a response.

Wrong

```js
res.status(200).json({

    message: "Done"

});

next();
```

A request should either:

- Send a response

OR

- Call `next()`

Not both.

---

### Mistake 3

Putting business logic inside middleware.

Middleware should handle common tasks like:

- Authentication
- Validation
- Logging

Business logic belongs in controllers.

---

# Best Practices

- Keep middleware focused on one responsibility.
- Reuse middleware whenever possible.
- Call `next()` when appropriate.
- Organize middleware inside a dedicated `middlewares` folder.
- Avoid placing business logic in middleware.

---

# Summary

- Middleware executes before the controller.
- It has access to `req`, `res`, and `next`.
- `next()` passes control to the next middleware or controller.
- Middleware is commonly used for authentication, validation, logging, and parsing requests.
- Express applications often use multiple middleware functions to process a request before reaching the controller.
