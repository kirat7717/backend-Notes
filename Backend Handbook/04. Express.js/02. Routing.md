# Routing

---

# Why

When a client sends a request, the server needs to decide **which code should handle that request**.

For example:

- Register User
- Login User
- Get Blogs
- Update Blog
- Delete Blog

If every request went to the same function, the application would become difficult to manage.

Routing solves this problem.

---

# What is Routing?

Routing is the process of matching an incoming request to the correct controller or function.

A route consists of:

- HTTP Method
- URL (Path)
- Handler Function

Example

```js
app.get("/blogs", (req, res) => {

});
```

Here,

- GET → HTTP Method
- /blogs → Route Path
- Function → Route Handler

---

# Route Syntax

```js
app.METHOD(PATH, HANDLER);
```

Example

```js
app.get("/", (req, res) => {

    res.send("Server Running");

});
```

---

# GET Route

Used to retrieve data.

```js

});
```

---

# POST Route

Used to create data.

```js

});
```

---

# PATCH Route

Used to update selected fields.

```js
app.patch("/blogs/:id", (req, res) => {

});
```

---

# DELETE Route

Used to remove data.

```js
app.delete("/blogs/:id", (req, res) => {

});
```

---

# Route Parameters

Routes can contain dynamic values.

Example

```js
app.get("/blogs/:id", (req, res) => {

});
```

Request

```text
GET /blogs/123
```

Access

```js
req.params.id
```

Output

```text
123
```

---

# Query Parameters

Used for filtering, searching, sorting, and pagination.

Example

```text
GET /blogs?page=1&limit=10
```

Access

```js
req.query.page

req.query.limit
```

# Express Router

As applications grow, writing all routes inside one file becomes difficult.

Express provides **Router** to organize routes.

Example

```js
import { Router } from "express";

const router = Router();
```

Create routes

```js
router.get("/blogs", getBlogs);

router.post("/blogs", createBlog);
```

Export

```js
export default router;
```

---

# Connecting Router

In `app.js`

```js
import blogRouter from "./routes/blogRoutes.js";

app.use("/api/v1", blogRouter);
```

Now

```js
router.get("/blogs");
```

becomes

```text
GET /api/v1/blogs
```

---

# Route Organization

Instead of

```text
index.js

↓

100+ Routes
```

Use

```text
routes/

├── userRoutes.js

├── blogRoutes.js

├── commentRoutes.js
```

Each file manages one resource.

# Common Mistakes

### Mistake 1

Writing all routes inside one file.

As the project grows, split routes into multiple router files.

---

### Mistake 2

Putting business logic inside routes.

Wrong

```js
router.post("/register", async (req, res) => {

    // 100 lines of logic

});
```

Correct

```js
router.post("/register", registerUser);
```

Business logic belongs in controllers.

---

### Mistake 3

Using the wrong HTTP method.

Wrong

```js
router.get("/createBlog");
```

Correct

```js
router.post("/blogs");
```

---

### Mistake 4

Not using route parameters.

Wrong

```text
/blog?id=123
```

For identifying a single resource, prefer

```text
/blogs/:id
```

---

# Best Practices

- Keep routes clean and simple.
- Store business logic in controllers.
- Group related routes into separate files.
- Use RESTful endpoint naming.
- Use route parameters for resource IDs.
- Use query parameters for filtering and pagination.

---

# Summary

- Routing connects incoming requests to the correct controller.
- Every route consists of an HTTP method, a URL, and a handler.
- Express Router helps organize routes into separate files.
- Routes should only define endpoints, not business logic.
- A well-organized routing structure improves scalability and maintainability.
