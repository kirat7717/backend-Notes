# 📘 Learning Philosophy

Every topic should follow this cycle.

```text
Learn

↓

Understand

↓

Build

↓

Debug

↓

Apply

↓

Document

↓

Revise
```

Learning without building is incomplete.

Building without understanding creates confusion.

Notes without revision lose their value.

All three are necessary.

---

# 🧠 How I Should Learn

For every new concept, I should answer these questions.

1. Why does it exist?
2. What problem does it solve?
3. How does it work?
4. Where is it used in real projects?
5. Can I implement it myself?

If I cannot answer these questions, I have not fully understood the topic.
