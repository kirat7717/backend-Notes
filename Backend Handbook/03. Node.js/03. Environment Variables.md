# Environment Variables

---

# Why

Every application has configuration values that may change depending on where it is running.

Examples include:

- Server Port
- Database URL
- JWT Secret
- Cloudinary API Keys
- Email Credentials

Hardcoding these values inside the source code is insecure and difficult to manage.

Environment Variables solve this problem.

---

# What are Environment Variables?

Environment Variables are values stored **outside the source code**.

The application reads these values while running.

This allows developers to change configuration without modifying the code.

---

# Why Use Environment Variables?

Environment Variables help to:

- Protect sensitive information
- Make applications configurable
- Support multiple environments
- Keep source code clean
- Prevent secrets from being pushed to GitHub

---

```text

EMAIL
```

---

# The `.env` File

Environment Variables are commonly stored inside a `.env` file.

Example

```env
PORT=3000

MONGO_URI=mongodb://127.0.0.1:27017/blogdb

JWT_SECRET=mySecretKey

CLOUDINARY_CLOUD_NAME=mycloud

CLOUDINARY_API_KEY=123456

CLOUDINARY_API_SECRET=abcdef
```

The `.env` file should be placed in the project's root directory.

---

# dotenv Package

Node.js cannot read `.env` files by itself.

The `dotenv` package loads environment variables into the application.

Install

```bash
npm install dotenv
```

---

# Loading Environment Variables

Import dotenv

```js
import dotenv from "dotenv";
```

Configure dotenv

```js
dotenv.config();
```

Now all variables inside `.env` become available through `process.env`.

---

# Accessing Environment Variables

Syntax

```js
process.env.VARIABLE_NAME
```

Example

```js
const PORT = process.env.PORT;
```

```js
const MONGO_URI = process.env.MONGO_URI;
```

---

# Using Environment Variables

Starting Server

```js
const PORT = process.env.PORT;

app.listen(PORT);
```

---

Connecting Database

```js
mongoose.connect(process.env.MONGO_URI);
```

---

Creating JWT

```js
jwt.sign(payload, process.env.JWT_SECRET);
```

---

Cloudinary Configuration

```js
cloudinary.config({

    cloud_name: process.env.CLOUDINARY_CLOUD_NAME,

    api_key: process.env.CLOUDINARY_API_KEY,

    api_secret: process.env.CLOUDINARY_API_SECRET

});
```

---

# process.env

`process` is a global object provided by Node.js.

`process.env` is an object containing all environment variables.

Example

```js
console.log(process.env);
```

Accessing one variable

```js
console.log(process.env.PORT);
```

---

# .env.sample

Projects should include a `.env.sample` file.

Example

```env
PORT=

MONGO_URI=

JWT_SECRET=

CLOUDINARY_CLOUD_NAME=

CLOUDINARY_API_KEY=

CLOUDINARY_API_SECRET=
```

This tells other developers which variables are required without exposing actual values.

---

# .gitignore

The `.env` file should never be uploaded to GitHub.

Example

```text
node_modules/

.env
```

This keeps secrets private.

---

# Development vs Production

Development

```text
PORT=3000

MONGO_URI=Local MongoDB
```

Production

```text
PORT=10000

MONGO_URI=MongoDB Atlas
```

The code remains the same.

Only environment variables change.

# Common Mistakes

### Mistake 1

Hardcoding secrets.

Wrong

```js
const JWT_SECRET = "mySecret";
```

Correct

```js
const JWT_SECRET = process.env.JWT_SECRET;
```

---

### Mistake 2

Uploading `.env` to GitHub.

Always add

```text
.env
```

to `.gitignore`.

---

### Mistake 3

Forgetting to call

```js
dotenv.config();
```

Without it,

```js
process.env.PORT
```

will be

```text
undefined
```

---

### Mistake 4

Using incorrect variable names.

Wrong

```js
process.env.port
```

Correct

```js
process.env.PORT
```

Environment variable names are case-sensitive.

---

# Best Practices

- Store all sensitive information inside `.env`.
- Never hardcode API keys or secrets.
- Add `.env` to `.gitignore`.
- Provide a `.env.sample` file.
- Use meaningful environment variable names.
- Access variables through `process.env`.

---

# Summary

- Environment Variables store configuration outside the source code.
- The `.env` file holds these variables during development.
- The `dotenv` package loads `.env` into `process.env`.
- Sensitive information such as database URLs, JWT secrets, and API keys should always be stored in environment variables.
- Using environment variables makes applications more secure, flexible, and production-ready.
