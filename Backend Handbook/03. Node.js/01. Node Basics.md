# Node Basics

---

# Why

JavaScript was originally created to run only inside web browsers.

It was mainly used for:

- Making web pages interactive
- Manipulating HTML and CSS
- Handling user events

However, JavaScript could not:

- Create web servers
- Connect to databases
- Read or write files
- Access the operating system

This limitation meant developers had to use different languages for backend development.

Node.js solved this problem by allowing JavaScript to run outside the browser.

---

# What is Node.js?

Node.js is a **JavaScript Runtime Environment** that allows JavaScript code to execute outside the browser.

It is built on Google's **V8 JavaScript Engine**, which compiles JavaScript into machine code for fast execution.

Node.js provides additional APIs that allow JavaScript to interact with the operating system.

---

# What is a Runtime Environment?

A Runtime Environment is software that provides everything needed to execute a program.

For JavaScript, the runtime provides:

- JavaScript Engine
- Memory Management
- File System Access
- Networking
- Timers
- Operating System APIs

Without a runtime, JavaScript code cannot execute.

# V8 JavaScript Engine

Node.js uses Google's **V8 Engine**.

The V8 Engine is responsible for:

- Reading JavaScript code
- Compiling it into Machine Code
- Executing the Machine Code

Flow

```text
JavaScript Code

↓

V8 Engine

↓

Machine Code

↓

CPU Executes
```

This is one of the reasons Node.js is fast.

# Node.js vs Browser JavaScript

| Browser JavaScript | Node.js |
|--------------------|---------|
| Runs in Browser | Runs Outside Browser |
| Access to DOM | No DOM |
| Used for Frontend | Used for Backend |
| Cannot Read Local Files | Can Read Files |
| Cannot Create Servers | Can Create Servers |
| Browser APIs | Node APIs |

# Common Misconceptions

### Node.js is a Programming Language

❌ Incorrect

JavaScript is the programming language.

Node.js is the runtime environment.

---

### Node.js is a Framework

❌ Incorrect

Express.js is a framework.

Node.js provides the runtime on which Express runs.

---

### Node.js Replaces JavaScript

❌ Incorrect

Node.js extends JavaScript by allowing it to run outside the browser.

# Best Practices

- Use the latest stable LTS version of Node.js.
- Keep Node.js updated.
- Use npm to manage packages.
- Organize projects using a modular folder structure.
- Store configuration in environment variables.

---

# Summary

- Node.js is a JavaScript Runtime Environment.
- It allows JavaScript to run outside the browser.
- It is built on Google's V8 JavaScript Engine.
- Node.js provides access to operating system features such as files, networking, and processes.
- It forms the foundation of modern JavaScript backend development.
- Express.js and other backend frameworks run on top of Node.js.
