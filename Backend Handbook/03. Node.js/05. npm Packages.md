# npm & package.json

---

# Why

While building backend applications, we often need additional functionality such as:

- Express.js
- Mongoose
- JWT
- bcrypt
- Multer
- Cloudinary

Instead of writing these libraries ourselves, we install packages created by other developers.

Node.js provides **npm (Node Package Manager)** to install and manage these packages.

---

# What is npm?

npm (Node Package Manager) is the default package manager for Node.js.

It allows developers to:

- Install packages
- Remove packages
- Update packages
- Manage project dependencies

npm is automatically installed when Node.js is installed.

---

# What is a Package?

A package is reusable code written by other developers.

Examples

- express
- mongoose
- jsonwebtoken
- bcrypt
- multer
- dotenv

Instead of building everything from scratch, we install packages using npm.

---

# What is package.json?

`package.json` is the main configuration file of a Node.js project.

It stores information such as:

- Project name
- Version
- Description
- Scripts
- Dependencies
- Dev Dependencies
- Module type

Every Node.js project should have one.

---

# Creating package.json

Initialize a project

```bash
npm init
```

or

```bash
npm init -y
```

The `-y` flag creates `package.json` using default values.

---

# Example package.json

```json
{
  "name": "blog-api",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "nodemon src/index.js",
    "start": "node src/index.js"
  },
  "dependencies": {
    "express": "^5.1.0"
  }
}
```

---

# Installing Packages

Install a package

```bash
npm install express
```

Short form

```bash
npm i express
```

---

# Installing Multiple Packages

```bash
npm install express mongoose dotenv
```

---

# Installing Development Packages

Development packages are only needed while developing.

Example

```bash
npm install --save-dev nodemon
```

Short form

```bash
npm i -D nodemon
```

---

# Dependencies

Packages required for the application to run.

Example

```json
"dependencies": {

    "express": "^5.1.0",

    "mongoose": "^8.0.0"

}
```

---

# Dev Dependencies

Packages used only during development.

Example

```json
"devDependencies": {

    "nodemon": "^3.0.0"

}
```

---

# package-lock.json

When packages are installed,

npm automatically creates

```text
package-lock.json
```

Its purpose is to:

- Lock exact package versions
- Ensure every developer installs the same versions
- Improve installation speed

Do not edit this file manually.

---

# node_modules

Every installed package is stored inside

```text
node_modules/
```

Example

```text
node_modules/

↓

express/

↓

mongoose/

↓

bcrypt/
```

This folder can become very large.

It should **never** be uploaded to GitHub.

---

# Scripts

Scripts allow us to run commands using npm.

Example

```json
"scripts": {

    "start": "node src/index.js",

    "dev": "nodemon src/index.js"

}
```

Run them using

```bash
npm run start
```

or

```bash
npm run dev
```

# Common Mistakes

### Mistake 1

Deleting `package.json`.

Without it, npm cannot manage the project.

---

### Mistake 2

Uploading `node_modules` to GitHub.

Always add

```text
node_modules/
```

to `.gitignore`.

---

### Mistake 3

Editing `package-lock.json` manually.

npm manages this file automatically.

---

### Mistake 4

Installing development tools as normal dependencies.

Tools like Nodemon should be installed as **Dev Dependencies**.

---

# Best Practices

- Always initialize a project using `npm init -y`.
- Keep `package.json` updated.
- Never upload `node_modules`.
- Use Dev Dependencies for development tools.
- Use meaningful script names.
- Commit both `package.json` and `package-lock.json` to GitHub.

---

# Summary

- npm is the package manager for Node.js.
- Packages provide reusable functionality.
- `package.json` manages project configuration and dependencies.
- `node_modules` stores installed packages.
- `package-lock.json` locks package versions.
- npm scripts simplify running common project commands.
