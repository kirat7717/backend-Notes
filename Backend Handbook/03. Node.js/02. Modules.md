# Modules

---

# Why

As applications grow, writing all code in a single file becomes difficult to manage.

Problems with a single file:

- Difficult to read
- Difficult to debug
- Difficult to maintain
- Difficult to reuse code

Modules solve this problem by allowing us to divide our application into multiple files, each responsible for a specific task.

---

# What is a Module?

A module is simply a JavaScript file.

Every JavaScript file in Node.js is considered a module.

Modules allow code to be:

- Exported
- Imported
- Reused

Example

```text
userController.js

тЖУ

Exports registerUser()

тЖУ

userRoutes.js

тЖУ

Imports registerUser()
```

# Types of Modules

Node.js supports two module systems.

- CommonJS (Older)
- ES Modules (Modern)

Modern backend applications commonly use **ES Modules**.

---

# ES Modules

ES Modules use:

- `export`
- `import`

Example

```js
export const appName = "Blog API";
```

Import

```js
import { appName } from "./config.js";
```

---

# Export

The `export` keyword makes variables, functions, or objects available to other files.

Example

```js
export const username = "Kirat";
```

---

Exporting Functions

```js
export const registerUser = (req, res) => {

};
```

---

Exporting Multiple Values

```js
export const registerUser = () => {

};

export const loginUser = () => {

};
```

---

# Import

The `import` keyword is used to access exported code.

Syntax

```js
import { variable } from "./file.js";
```

Example

```js
import { registerUser } from "../controllers/userController.js";
```

---

# Importing Multiple Values

```js
import {
    registerUser,
    loginUser
} from "../controllers/userController.js";
```

---

# Default Export

A file can have one default export.

Example

```js
const connectDB = () => {

};

export default connectDB;
```

Import

```js
import connectDB from "./db/index.js";
```

Notice that default imports do not use curly braces.

---

# Named Export vs Default Export

## Named Export

Export

```js
export const registerUser = () => {

};
```

Import

```js
import { registerUser } from "./userController.js";
```

---

## Default Export

Export

```js
export default connectDB;
```

Import

```js
import connectDB from "./db/index.js";
```

---

# Enabling ES Modules

To use ES Modules, add the following in `package.json`.

```json
{
    "type": "module"
}
```

Without this configuration, Node.js uses the CommonJS module system.

---

# File Extensions

When importing local files, include the `.js` extension.

Correct

```js
import connectDB from "./db/index.js";
```

Incorrect

```js
import connectDB from "./db/index";
```

# Common Mistakes

### Mistake 1

Forgetting `"type": "module"` in `package.json`.

Result

```text
Cannot use import statement outside a module
```

---

### Mistake 2

Using curly braces with a default export.

Wrong

```js
import { connectDB } from "./db.js";
```

Correct

```js
import connectDB from "./db.js";
```

---

### Mistake 3

Forgetting the `.js` extension.

Wrong

```js
import router from "./routes/userRoutes";
```

Correct

```js
import router from "./routes/userRoutes.js";
```

---

### Mistake 4

Importing a named export without curly braces.

Wrong

```js
import registerUser from "./userController.js";
```

Correct

```js
import { registerUser } from "./userController.js";
```

---

# Best Practices

- Keep one responsibility per module.
- Use meaningful file names.
- Group related functions together.
- Use named exports when exporting multiple values.
- Use default exports when a file has one primary responsibility.
- Organize modules into folders such as `controllers`, `routes`, `models`, `middlewares`, and `utils`.

---

# Summary

- A module is a JavaScript file.
- Modules make applications organized and reusable.
- `export` shares code with other files.
- `import` uses exported code.
- ES Modules are the modern module system in Node.js.
- Modular architecture makes backend applications easier to build, test, and maintain.
