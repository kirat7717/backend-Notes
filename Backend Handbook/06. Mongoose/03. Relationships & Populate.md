# Relationships & Populate

---

# Why

Real-world applications contain related data.

Examples:

- A Blog belongs to a User.
- A Comment belongs to a Blog.
- An Order belongs to a Customer.
- A Product belongs to a Category.

Instead of storing the complete related document, MongoDB stores a reference using an **ObjectId**.

Mongoose uses `ref` and `populate()` to work with these relationships.

---

# What is a Relationship?

A relationship connects one document to another document.

Instead of duplicating data, MongoDB stores the **ObjectId** of the related document.

Example

```text
User

↓

ObjectId

↓

Blog
```

---

# Why Store ObjectId?

Suppose we store the entire user inside every blog.

```json
{
  "title": "Learning Node.js",
  "author": {
    "username": "Kirat",
    "email": "kirat@gmail.com"
  }
}
```

Now imagine the user changes their email.

Every blog document would also need to be updated.

Instead, MongoDB stores only the user's ObjectId.

```json
{
  "title": "Learning Node.js",
  "authorId": "687a8d1..."
}
```

This avoids duplicate data.

---

# Creating a Relationship

Inside the Blog Schema

```js
authorId: {

    type: mongoose.Schema.Types.ObjectId,

    ref: "User",

    required: true

}
```

Explanation

- `ObjectId` → Stores the user's unique ID.
- `ref` → Tells Mongoose this ID belongs to the **User** model.

```text

↓

User Document

↓

```

# Before Populate

User Collection

```json
{
  "_id": "111",
  "username": "Kirat",
  "email": "kirat@gmail.com"
}
```

Blog Collection

```json
{
  "_id": "222",
  "title": "Learning Express",
  "authorId": "111"
}
```

Notice that only the ObjectId is stored.

---

# What is Populate?

`populate()` replaces the stored ObjectId with the complete related document.

Instead of returning

```json
{
  "authorId": "111"
}
```

It returns

```json
{
  "authorId": {
    "_id": "111",
    "username": "Kirat",
    "email": "kirat@gmail.com"
  }
}
```

---

# Syntax

```js
.populate("fieldName")
```

Example

```js
const blogs = await Blog.find().populate("authorId");
```

---

# Selecting Specific Fields

Sometimes we don't want every field.

Example

```js
const blogs = await Blog.find().populate("authorId", "username email");
```

Only

```text
username

email
```

will be returned.

Password will not.

---

# Real Project Example

Blog Schema

```js
authorId: {

    type: mongoose.Schema.Types.ObjectId,

    ref: "User"

}
```

Controller

```js
const blogs = await Blog.find().populate("authorId", "username email");
```

Result

```json
{
  "title": "Learning Mongoose",
  "authorId": {
    "username": "Kirat",
    "email": "kirat@gmail.com"
  }
}
```

---

# Multiple Populate

You can populate multiple relationships.

Example

```js
const blogs = await Blog.find()

  .populate("authorId")

  .populate("categoryId");
```

---

# Populate with Sorting

```js
const blogs = await Blog.find()

  .populate("authorId", "username email")

  .sort({
    createdAt: -1,
  });
```

This is exactly what we used in our Blog Project.

# Common Mistakes

### Mistake 1

Using `populate()` without `ref`.

Wrong

```js
authorId: mongoose.Schema.Types.ObjectId;
```

Correct

```js
authorId: {

    type: mongoose.Schema.Types.ObjectId,

    ref: "User"

}
```

---

### Mistake 2

Expecting `populate()` to work automatically.

You must call

```js
.populate()
```

when querying.

---

### Mistake 3

Returning sensitive fields.

Wrong

```js
.populate("authorId")
```

This may also return the password.

Better

```js
.populate(
    "authorId",
    "username email"
)
```

---

### Mistake 4

Thinking `populate()` changes the database.

It only changes the data returned by the query.

The stored document still contains only the ObjectId.

---

# Best Practices

- Store relationships using ObjectId.
- Always use `ref` for related documents.
- Use `populate()` only when related data is needed.
- Select only required fields while populating.
- Avoid exposing sensitive fields like passwords.

---

# Summary

- Relationships connect documents using ObjectId.
- `ref` tells Mongoose which model the ObjectId belongs to.
- `populate()` replaces the ObjectId with the related document.
- Use field selection to return only necessary data.
- Relationships reduce duplicate data and keep the database organized.
