# Query Helpers

---

# Why

Sometimes we perform the same query repeatedly.

Example

```js
await Blog.find({
    isPublished: true
});
```

Again

```js
await Blog.find({
    isPublished: true
});
```

Again

```js
await Blog.find({
    isPublished: true
});
```

Repeating the same query everywhere makes the code harder to maintain.

Mongoose provides **Query Helpers** to solve this problem.

---

# What are Query Helpers?

Query Helpers are custom methods that you define on a Schema.

They allow you to create reusable queries.

Instead of writing the same filter repeatedly, you can call a custom helper method.

---

---

# Defining a Query Helper

Syntax

```js
schema.query.helperName = function () {

    return this.find(...);

};
```

---

# Example

```js
blogSchema.query.published = function () {

    return this.find({

        isPublished: true

    });

};
```

Now

```js
await Blog.find().published();
```

returns only published blogs.

---

# Using Multiple Conditions

Example

```js
blogSchema.query.byAuthor = function (authorId) {

    return this.find({

        authorId

    });

};
```

Usage

```js
const blogs = await Blog.find()

.byAuthor(userId);
```

---

# Chaining Query Helpers

Query Helpers can be chained.

Example

```js
blogSchema.query.published = function () {

    return this.find({

        isPublished: true

    });

};
```

Usage

```js
const blogs = await Blog.find()

.published()

.sort({

    createdAt: -1

});
```

---

# Query Helper vs Normal Query

Without Query Helper

```js
const blogs = await Blog.find({

    isPublished: true

});
```

---

With Query Helper

```js
const blogs = await Blog.find()

.published();
```

The second approach is cleaner and reusable.

# Real Project Example

Suppose our Blog Schema contains

```js
isPublished: {

    type: Boolean,

    default: false

}
```

Create Helper

```js
blogSchema.query.published = function () {

    return this.find({

        isPublished: true

    });

};
```

Controller

```js
const blogs = await Blog.find()

.published();
```

---

```js

```

Controller

```js
const blogs = await Blog.find()

```

# Query Helpers vs Static Methods

Query Helper

```js
Blog.find()

.published();
```

Works on queries.

---

Static Method

```js
Blog.published();
```

Works directly on the Model.

They solve similar problems but are used differently.

---

# Common Mistakes

### Mistake 1

Forgetting to return the query.

Wrong

```js
blogSchema.query.latest = function () {

    this.sort({

        createdAt: -1

    });

};
```

Correct

```js
blogSchema.query.latest = function () {

    return this.sort({

        createdAt: -1

    });

};
```

---

### Mistake 2

Thinking Query Helpers modify the database.

They only modify the query.

No document is changed.

---

### Mistake 3

Using Query Helpers for one-time queries.

If a query is only used once, writing it directly is usually simpler.

---

# Best Practices

- Use Query Helpers for frequently repeated queries.
- Give helper methods meaningful names.
- Keep Query Helpers small and focused.
- Combine Query Helpers with sorting, limiting, and pagination when needed.
- Use them to improve code readability.

---

# Summary

- Query Helpers are custom query methods defined on a Schema.
- They help avoid repeating the same query logic.
- Query Helpers make controllers cleaner and easier to maintain.
- They work on query objects and support method chaining.
- They are useful for reusable filters such as published blogs, active users, or latest records.
