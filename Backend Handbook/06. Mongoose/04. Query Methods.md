# Query Methods

---

# Why

A Model represents a MongoDB collection.

To interact with the collection, Mongoose provides many built-in methods.

These methods allow us to:

- Create documents
- Find documents
- Update documents
- Delete documents

Instead of writing raw MongoDB queries, we use Mongoose Query Methods.

---

# What are Query Methods?

Query Methods are built-in functions provided by Mongoose Models to perform CRUD operations on MongoDB.

Example

```js
await User.find();
```

Here,

```text
find()
```

is a Query Method.

---

# Query Method Flow

```text
Controller

↓

```

---

# create()

Creates a new document.

Syntax

```js
await Model.create(data);
```

Example

```js
await User.create({

    username: "Kirat",

    email: "kirat@gmail.com"

});
```

Returns

```text
Newly Created Document
```

---

# find()

Returns all matching documents.

Syntax

```js
await Model.find(filter);
```

Example

```js
const blogs = await Blog.find();
```

Returns

```text
Array of Documents
```

---

# findOne()

Returns the first matching document.

Example

```js
const user = await User.findOne({

    email

});
```

Returns

```text
Single Document
```

or

```text
null
```

---

# findById()

Finds a document using its `_id`.

Example

```js
const blog = await Blog.findById(blogId);
```

Returns

```text
Document
```

or

```text
null
```

---

# updateOne()

Updates one document.

Example

```js
await Blog.updateOne(

    {

        _id: blogId

    },

    {

        $set: {

            title: "Updated Title"

        }

    }

);
```

Returns update information.

It does **not** return the updated document.

---

# findOneAndUpdate()

Finds, updates, and optionally returns the updated document.

Example

```js
const blog = await Blog.findOneAndUpdate(

    {

        _id: blogId

    },

    {

        $set: updateData

    },

    {

        new: true

    }

);
```

Returns

```text
Updated Document
```

---

# deleteOne()

Deletes one matching document.

Example

```js
await Blog.deleteOne({

    _id: blogId

});
```

---

# findByIdAndDelete()

Deletes a document using its `_id`.

Example

```js
await Blog.findByIdAndDelete(blogId);
```

Returns the deleted document.

---

# exists()

Checks whether a document exists.

Example

```js
const userExists = await User.exists({

    email

});
```

Returns

```text
true-like object
```

or

```text
null
```

Useful when you only need to know whether a document exists.

---

# countDocuments()

Returns the total number of matching documents.

Example

```js
const totalBlogs = await Blog.countDocuments();
```

---

# sort()

Sorts query results.

Newest First

```js
const blogs = await Blog.find()

.sort({

    createdAt: -1

});
```

Oldest First

```js
.sort({

    createdAt: 1

});
```

---

# limit()

Limits the number of documents.

Example

```js
const blogs = await Blog.find()

.limit(5);
```

---

# skip()

Skips documents.

Useful for pagination.

Example

```js
const blogs = await Blog.find()

.skip(10)

.limit(10);
```

---

# populate()

Loads related documents.

Example

```js
const blogs = await Blog.find()

.populate(

    "authorId",

    "username email"

);
```

---

# Chaining Query Methods

Multiple query methods can be chained together.

Example

```js
const blogs = await Blog.find()

.populate(

    "authorId",

    "username email"

)

.sort({

    createdAt: -1

})

.limit(10);
```

# Choosing the Right Method

| Method | Use Case |
|---------|----------|
| create() | Create one document |
| find() | Get multiple documents |
| findOne() | Get one matching document |
| findById() | Find by `_id` |
| updateOne() | Update without returning document |
| findOneAndUpdate() | Update and return document |
| deleteOne() | Delete one matching document |
| findByIdAndDelete() | Delete by `_id` |
| exists() | Check if a document exists |
| countDocuments() | Count documents |
| populate() | Load related documents |

# Common Mistakes

### Mistake 1

Using `find()` when only one document is expected.

Wrong

```js
const user = await User.find({

    email

});
```

Correct

```js
const user = await User.findOne({

    email

});
```

---

### Mistake 2

Using `updateOne()` when updated data is required.

Instead use

```js
findOneAndUpdate()
```

with

```js
new: true
```

---

### Mistake 3

Forgetting `await`.

Wrong

```js
const blogs = Blog.find();
```

Correct

```js
const blogs = await Blog.find();
```

---

### Mistake 4

Using `findById()` with non-ObjectId values.

`findById()` should only receive a valid MongoDB ObjectId.

---

# Best Practices

- Use the most appropriate query method.
- Prefer `findOne()` when expecting one document.
- Use `findById()` for `_id` searches.
- Use method chaining for sorting, pagination, and population.
- Always handle cases where queries return `null`.

---

# Summary

- Query Methods are built-in functions provided by Mongoose Models.
- They simplify CRUD operations.
- Common methods include `create()`, `find()`, `findOne()`, `findById()`, `findOneAndUpdate()`, and `findByIdAndDelete()`.
- Methods can be chained with `sort()`, `limit()`, `skip()`, and `populate()`.
- Choosing the correct query method leads to cleaner and more efficient backend code.
