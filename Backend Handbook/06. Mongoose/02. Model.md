# Model

---

# Why

A Schema only defines the structure of a document.

However, a Schema cannot:

- Save data
- Read data
- Update data
- Delete data

To perform database operations, Mongoose provides a **Model**.

A Model acts as the bridge between our application and MongoDB.

---

# What is a Model?

A Model is a JavaScript object created from a Schema.

It provides methods to interact with a MongoDB collection.

Examples:

- Create documents
- Find documents
- Update documents
- Delete documents

---

# Model Flow

```text
Schema

↓

Model

↓

MongoDB Collection

↓

Documents
```

---

# Creating a Model

Syntax

```js
const ModelName = mongoose.model(
    "CollectionName",
    schema
);
```

Example

```js
const User = mongoose.model(
    "User",
    userSchema
);
```

---

# Exporting the Model

```js
const User = mongoose.model(
    "User",
    userSchema
);

export default User;
```

Now the model can be used anywhere in the project.

---

# Importing the Model

```js
import User from "../models/user.model.js";
```

Now we can perform database operations.

---

# What Can a Model Do?

A Model provides methods such as:

- create()
- find()
- findOne()
- findById()
- updateOne()
- findOneAndUpdate()
- deleteOne()
- findByIdAndDelete()

All CRUD operations are performed through the Model.

---

# Model vs Schema

| Schema | Model |
|---------|--------|
| Blueprint | Working Object |
| Defines Structure | Interacts with Database |
| No CRUD Operations | CRUD Operations |
| One per Collection | Used Throughout Project |

---

# Relationship Between Schema and Model

```text
Schema

↓

Defines Structure

↓

Model

↓

Creates Collection

↓

Stores Documents
```

Without a Schema, a Model cannot be created.

Without a Model, database operations cannot be performed.

# Real Project Example

## User Schema

```js
const userSchema = new mongoose.Schema({

    username: String,

    email: String,

    password: String

});
```

---

## User Model

```js
const User = mongoose.model(
    "User",
    userSchema
);

export default User;
```

Now

```js
await User.create();

await User.find();

await User.findOne();

await User.findById();
```

all become available.

```js

await Blog.findOne();
```

# Model Creates Collection

Suppose we create

```js
const User = mongoose.model(
    "User",
    userSchema
);
```

MongoDB automatically creates the collection

```text
users
```

Notice:

Model Name

```text
User
```

Collection Name

```text
users
```

Mongoose automatically converts it to lowercase and pluralizes it.

# Common Mistakes

### Mistake 1

Trying to perform CRUD operations on a Schema.

Wrong

```js
userSchema.find();
```

Correct

```js
User.find();
```

Only Models perform database operations.

---

### Mistake 2

Creating multiple models for the same collection.

One collection should have one model.

---

### Mistake 3

Forgetting to export the model.

Without exporting,

other files cannot use it.

---

### Mistake 4

Confusing Model name with Collection name.

Model

```text
User
```

Collection

```text
users
```

Mongoose automatically pluralizes the collection name.

---

# Best Practices

- Create one Model for each Schema.
- Keep Models inside the `models` folder.
- Use PascalCase for Model names.
- Export Models immediately after creating them.
- Perform all database operations using Models.

---

# Summary

- A Model is created from a Schema.
- Models communicate with MongoDB.
- CRUD operations are performed through Models.
- Every collection should have its own Model.
- Schemas define structure, while Models perform database operations.
- Models act as the bridge between the application and MongoDB.
